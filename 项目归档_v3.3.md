## 二、各模块完整功能

### 1. KOS 任务中心

**素材五分类(管理员上传,2026-07-14 起新库)**:主图 / 横版近景 / 横版远景 / 竖版近景 / 竖版远景,分开上传、分开计数;旧三分类库(主图/2合1/4合1)按旧逻辑发完为止,显示「旧版三分类库」标签,勿混传新分类素材。浏览器预签名直传 COS;进后台自动与桶对账(60 秒节流,手动「刷新桶内数量」实时对账),读不到 COS 时红字提示原因。删库会一并清 COS 目录。

**出图规则(统一 3:4 = 1080×1440,不翻转不变形)**:
- 主图:单张 cover 居中裁成 3:4(随机抽取,可复用)。
- 2合1:横版近景 × 横版远景 随机配对,各裁成 1080×720 后上下拼(上下位置随机)。
- 4合1:竖版近景 2 张 + 竖版远景 2 张随机组合,各裁成 540×720 后田字 2×2 拼(四格位置随机)。
- 先把每张裁到精确尺寸再无缝拼接;开图按 EXIF 方向自动转正。

**容量与唯一性(v3.3 起数据库级保证)**:每份 = 主图×1(随机可复用)+ 一对横版(唯一)+ 一组四张竖版(唯一);五分类库总量 = min(横近×横远, C(竖近,2)×C(竖远,2))(需 横近≥1/横远≥1/竖近≥2/竖远≥2);旧三分类库 = min(C(横版,2), C(竖版,4))。领取走**单事务原子分配**(`claim_pack`:限额校验+选组合+落库一体,`BEGIN IMMEDIATE` 串行化)+ combo 唯一索引兜底;下载/出图失败自动回滚释放组合。全员周四同时领取不会再撞出重复素材或超领。

**业务闭环(领取页)**:本期口径(周三起点,周三→下周二)双卡(任务笔记 / 自发布笔记);领取 → 出 3 图(令牌下载)+ 分栏文案(各自一键复制,文案空时可「重新生成文案」);素材包内可「一键生成封面图」(AI 按本篇文案配字出候选,持久化保存,再来一批可换);**已领素材包跨设备同步**(换设备登录可见);「笔记链接」区上任务发布(仅进行中任务)/下自发布;回收链接只认小红书域名(**hostname 严格校验**,防钓鱼混入);自发布笔记登记单独计数;排行按「本期任务完成 + 本期自发布」。

**管理端**:建库、发任务(改名「任务创建」,进行中任务可「定向补发」+1 篇配额);数据看板 KOS 标签含:笔记排行(任务+自发布合并计数)、每日发布、素材库用量、任务笔记链接、**自发布笔记链接**(v3.3)、**本期任务完成情况看板**(v3.3:合并全部进行中任务,应完成/已完成/进行中/未开始人数 + 已发布篇数 + 完成率,三栏人员名单每人 x/N 篇,未开始排最前便于催办,支持按部门筛选联动)。

### 2. 小红书生成助手

- 选品牌 + 产品后:**一键生成文案** / **一键生成封面图**。
- **文案**:四类型;随机种草角度 + 变体 nonce + 失败重试;正文清洗;标签固定批次永不缺失。
- **封面**:上传照片 + 三段文字 → AI(Seedream 5.0 Pro)一次出 3 张候选(**尺寸自适应不拉伸原图**、无水印);一键导入智能兜底;18 套配色;照片主体不变;出图规则见第八节。

### 3. 销售问答助手

基于说明书 OCR + 卖点,三层防幻觉;答后附销售话术指引;售后/保修智能兜底(4000480500 热线)。

### 4. 产品资料库

COS 私有桶 + 后端预签名下载(短时效)+ 白名单(「价格与政策」永不外泄);后台可上传/删除/改名资料。

## 三、v3.3 本轮更新(2026-07-06,全面安全加固 + 看板增强)

源自一次全量代码审查(约 12000 行,报告见 `代码审查与优化建议_20260706.md`),分四批全部落地:

**第一批 · P0 安全**
1. **密钥回退链移除**:SESSION_SECRET / ID6_SALT 只认专用密钥,缺失**拒绝启动**(此前会静默回退到 COS/DeepSeek API Key 当签名密钥——API Key 泄露即可伪造管理员 Token);遗留哈希只读兼容防锁人;`.env.example` 补齐全部密钥说明。
2. **停用即拒**:`get_user` 仅在数据库故障时才回退 users.json;停用用户不能再靠旧名单复活登录。
3. **登录防枚举**:按 IP 双闸限流(全量 8/分 150/日,失败 5/分 60/日锁定,真实 IP 取 X-Forwarded-For)+ **管理员强制 id6**(普通员工过渡期宽松不变)+ 后台用户管理新增 id6「设置/重设」入口。
4. **前端 XSS 清零**:AI 文案/品牌产品名/用户名不再进内联 onclick(单引号与实体解码两类突破),全部改 data-* + 事件绑定;错误信息统一转义。
5. **错误不裸奔**:16 处接口兜底统一「journalctl 记 traceback + 前端只见通用文案」;排查线上问题看 `journalctl -u agent`。

**第二批 · P1 数据一致性**
KOS 领取事务化 + 唯一索引 + 失败回滚(见第二节);小红书链接 hostname 严格校验;admin 看板 is_admin/roleOf 判权统一;`_user` 请求体注入全站移除(身份一律走 Bearer token);qa 失败日志 `dir()`→`locals()` 修复。

**第三批 · 性能与并发**
`lib/db.py` 线程级 SQLite 连接复用(统一 WAL + synchronous=NORMAL + busy_timeout,看板请求从几十个连接降到 1 个);`lib/aigate.py` 全局 AI 并发闸(默认 4 路在途,排队 20 秒报忙,防线程耗尽);CORS 全站收敛(删 28 处 `*`);dev_server 模块缓存加锁 + 响应半写保护;每日配额原子计数(UPSERT RETURNING);PIL 句柄 with 化;备份 tmp+完整性校验+原子改名;COS 对账 60 秒节流;sys.path 判重。

**第四批 · 前端收口**
Chart.js 4.4.0 自托管(`assets/vendor/`,不再依赖 jsdelivr);index.html 删旧管理面板等约 330 行死代码/死 CSS;escapeHtml 与会话鉴权统一走 app.js(`App.escapeHtml` / `App.requireSession`);admin 登出补清 token。

## 四、技术栈与架构

| 层 | 选型 |
|---|---|
| 运行 | 多线程 Python `http.server`(`scripts/dev_server.py`,自动路由 `/api/<name>`→`api/<name>.py`,模块缓存加锁,启动强校验必需密钥)|
| 数据库 | 本地 SQLite `data/usage.db`;**连接统一走 `lib/db.py`(每线程复用,WAL + NORMAL + busy_timeout)**|
| 会话 | `lib/session.py`:HMAC-SHA256 无状态 Token(24h),密钥只认 `SESSION_SECRET` |
| 并发闸 | `lib/aigate.py`:全服 AI 在途请求上限(`AI_MAX_CONCURRENCY`,默认 4)|
| 限流 | `lib/ratelimit.py`:每人分钟/每日双闸(原子计数),登录/花名册另有按 IP 独立闸 |
| 对象存储 | 腾讯云 COS `mkt-1421994952`(ap-shanghai):下载走后端预签名 GET,上传走浏览器直传预签名 PUT |
| 文案 AI | DeepSeek 系(文案 / 封面字段 / 问答)|
| 视觉 AI | 豆包 **Seedream 5.0 Pro**(`doubao-seedream-5-0-pro-260628`,`DOUBAO_MODEL` 可覆盖);图生图,输出尺寸 `adaptive` 跟随输入图比例不拉伸(`DOUBAO_IMAGE_SIZE` 可覆盖)|
| 图像处理 | Pillow(KOS 拼图;句柄 with 管理)|
| 部署 | 腾讯云 Lighthouse + Nginx + systemd `agent.service` + Let's Encrypt |
| 前端 | 原生 HTML + 共享 `app.css`/`app.js`(escapeHtml/requireSession/fetch-token 补丁统一收口);Chart.js 自托管 |

**设计原则**:单进程、单 SQLite、配置驱动、绞杀者式渐进重构(全程不停服)。跨语句业务不变量一律事务 + 约束双保险;可用性兜底只兜「基础设施故障」,不兜「数据说没有」。

| 调 AI 并发/限流 | `.env`:`AI_MAX_CONCURRENCY` / `AI_QUEUE_TIMEOUT` / `RL_PER_MIN` / `RL_PER_DAY` |
| 数据库备份 | 自动(每日 03:00);手动 `python3 scripts/backup_db.py` |
| 部署 | Mac `git push`;服务器 `git pull https://ghproxy.net/https://github.com/AshinXuuu/xiaohongshu-JohnsonAgent.git main && sudo systemctl restart agent` |

**⚠️ 环境变量红线**:`SESSION_SECRET`、`ID6_SALT` 缺失服务拒绝启动;`ID6_SALT` 一旦有用户数据**不可更换**(已存哈希会全部失效);更换 `SESSION_SECRET` 代价仅为全员重新登录。

## 八、出图与文案规则要点

**KOS 拼图**(`lib/image_compose.py`):主图裁 3:4;2合1 上下拼;4合1 田字拼;先裁到单元格精确尺寸再拼;不翻转不变形;EXIF 转正。

**封面出图**(`api/cover-generate.py`):照片主体保持;三级文字(6-10 / 8-12 / 8-12 字)一字不差、规范简体;18 套配色按文案类型落池,每次随机 3 套并发;封面三段字由 `api/cover-fields.py` + `prompts/封面字段.txt` 提炼。
