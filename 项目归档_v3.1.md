## 一、当前形态:四模块 + 管理后台

工作台门户登录后,按配置顺序展示四个模块(卡片 + 各页顶栏标签栏同序):

| 顺序 | 模块 | 入口 | 作用 |
|---|---|---|---|

## 二、v3.1 本轮更新(2026-06-27 → 07-02)

### 1. 管理看板重构

- 新增**总览层**:活跃用户(今日/本周/时间段去重)、各模块使用量、按日趋势(Chart.js)、最活跃的人排行。
- **按模块分标签**:总览 / 文案助手 / 问答助手 / 资料库 / KOS 任务,各标签独立指标 + 图表。
- **时间范围切换** + 明细区可**搜索、翻页**(走 `/api/admin-events`)。
- 删除了原来的「全部」页;修掉了排行行间距(补 `.rank-list` 容器)。

### 2. 产品管理 = 卖点 + 资料双管理

产品管理页除原有「编辑卖点」外,新增**产品资料库内容管理**:显示某产品当前的资料文件、支持**上传 / 删除 / 改文件名**(浏览器直传 COS,预签名 PUT);全程写操作审计。

### 3. 操作日志(近 30 天)

- 新增 `lib/audit.py`(`admin_audit` 表),记录**用户管理、产品编辑、任务发布**三类后台写操作,自动清理 30 天前记录。
- 查看页 `/admin-log.html`:按类别筛选、按操作人搜索、翻页。

### 4. KOS 素材:浏览器直传 + 打开即与 COS 自动对账 ★

这是本轮改动最大的一块,解决"素材数量对不上、要进桶才看得到"的痛点:

- **直传上传**:每个素材库有「上传主图 / 上传拼图素材」按钮,支持一次多张 / 整个文件夹,浏览器**预签名 PUT 直传 COS**(不经服务器中转),传到 `主图/`、`可拼图/` 子目录。
- **打开页面即自动对账**(取代原「扫描登记」按钮):进 KOS 后台时,后台自动列 COS 目录 → 桶里有、库里没有的**自动登记**;库里有、桶里已删的**自动下架**。数量永远等于桶内实际,**管理员无需进 COS 控制台**。
- 顶部显示同步状态;**读不到 COS 时红字提示并附真实原因**(权限/密钥/地域等),不拿过时数字糊弄。
- 保留「↻ 刷新桶内数量」按钮供手动重查。
- **删除库**会一并清除 COS 目录下所有对象(不再是只删记录)。
- 素材按 **COS 子目录**(`主图/`、`可拼图/`)判定角色,兼容哈希文件名。

### 5. 数据库每日备份

- `scripts/backup_db.py`:SQLite 在线备份 API 打一致性快照到 `backups/`,自动清理 14 天前旧备份,只依赖 python3。
- 已配 cron(每日凌晨 3 点)。

## 四、技术栈与架构

| 层 | 选型 |
|---|---|
| 运行 | 单进程 Python `http.server`(`scripts/dev_server.py`,自动路由 `/api/<name>`→`api/<name>.py`)|
| 数据库 | 本地 SQLite `data/usage.db`(events / manuals / app_users / app_brands / app_products / kos_* / admin_audit)|
| 对象存储 | 腾讯云 COS `mkt-1421994952`(ap-shanghai):产品资料 + KOS 素材。下载走后端预签名 GET;上传走**浏览器直传预签名 PUT**(需子账号写权限 + 桶 CORS)|
| 文案 AI | DeepSeek(`deepseek-chat`)|
| 视觉 AI | 豆包(视觉识别 + Seedream 封面)|
| 图像处理 | Pillow(KOS 出图)|
| 部署 | 腾讯云 Lighthouse + Nginx + systemd `agent.service` + Let's Encrypt |
| 前端 | 原生 HTML + 共享 `app.css`/`app.js`(统一外壳,配置驱动,白标就绪)|

**设计原则**:单进程、单 SQLite、配置驱动、绞杀者式渐进重构(新模块用新方式建,老模块逐个迁,全程不停服)。

## 五、数据库表一览(usage.db)

| 表 | 作用 |
|---|---|
| `events` | 用量事件(login/generate/qa/cover/download…)|
| `manuals` / `manuals_files` | 问答 OCR 资料库 |
| `app_users` | 用户(从 users.json 播种,后台管理)|
| `app_brands` / `app_products` | 产品目录(从 products.json 播种,后台管理)|
| `library_files` | 产品资料文件清单(下载白名单)|
| `kos_libraries` | KOS 素材库(一产品可多批次,含 `cos_prefix`)|
| `kos_materials` | KOS 素材(主图/可拼图 → COS key)|
| `kos_tasks` | KOS 任务 |
| `kos_packs` | 领取记录 = 唯一组合消耗 + 完成回填 |
| `admin_audit` | 后台操作日志(用户/产品/任务,近 30 天)|

> `usage.db` 在 .gitignore 里,数据存服务器、不进 git。已配每日自动备份(`scripts/backup_db.py` + cron)。

## 七、日常维护(网页化,不再改文件)

| 要做的事 | 怎么做 |
|---|---|
| 加/停用用户、改角色 | 后台「用户管理」(只有超管能授予管理员角色)|
| 改卖点/增删产品 | 后台「产品管理」编辑卖点 |
| 加/删/改名 产品下载资料 | 后台「产品管理」→ 该产品资料区,直接上传/删除/改名(直传 COS)|
| 建 KOS 素材库/传素材 | 后台「KOS 任务」建库 →「上传主图/上传拼图素材」直传;**数量自动与桶对齐,无需扫描** |
| 发 KOS 任务 | 「KOS 任务」选库/部门/篇数/截止,容量不足会预警 |
| 看后台谁改了什么 | 后台「操作日志」(近 30 天,可按类别/操作人筛)|
| 数据库备份 | 已自动(每日 03:00 → `backups/`);手动 `python3 scripts/backup_db.py` |
| 部署 | `git pull && sudo systemctl restart agent` |
