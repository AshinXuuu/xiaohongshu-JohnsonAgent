## 五、☁️ 服务器:迁移线上数据库的身份证明文 → 哈希

服务器的 `data/usage.db` 是线上真实库,`git pull` **不会**动它,必须单独迁移一次。二选一:

**办法 A(推荐,最简单):进后台点一次「从名单重新导入」**
`git pull` 下来的 `data/users.json` 已经是哈希版,重导入会用哈希覆盖数据库。
> 登录后台 → 用户管理 → 「从名单重新导入」(仅超级管理员可点)。做完这一步即可,跳过办法 B。

**办法 B(命令行迁移,不想动后台就用这个):**

```bash
cd /项目路径
# 先备份线上库
cp data/usage.db "data/usage.db.bak.$(date +%Y%m%d%H%M%S)"

# 明文→哈希(读取 .env 里的 ID6_SALT)
python3 - <<'PY'
import os, sqlite3, sys
sys.path.insert(0, '.')
for line in open('.env', encoding='utf-8'):
    line = line.strip()
    if line and not line.startswith('#') and '=' in line:
        k, v = line.split('=', 1); os.environ.setdefault(k.strip(), v.strip())
from lib.idhash import hash_id6, is_hashed
con = sqlite3.connect('data/usage.db'); con.row_factory = sqlite3.Row
rows = con.execute('SELECT id, id_last6 FROM app_users').fetchall()
n = 0
for r in rows:
    raw = str(r['id_last6'] or '').strip()
    if raw and not is_hashed(raw):
        con.execute('UPDATE app_users SET id_last6=? WHERE id=?', (hash_id6(raw), r['id'])); n += 1
con.commit()
leak = con.execute("SELECT COUNT(*) FROM app_users WHERE id_last6<>'' AND id_last6 NOT LIKE 'h1$%'").fetchone()[0]
con.close()
print(f'已转哈希 {n} 条;明文残留 {leak} 条(应为 0)')
PY
```
