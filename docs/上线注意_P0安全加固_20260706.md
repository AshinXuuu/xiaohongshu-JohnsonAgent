# P0 安全加固上线注意(2026-07-06)

本批改动:密钥回退链移除 / 停用用户 JSON 兜底修复 / 登录限流 + 管理员强制 id6 / 前端 XSS 修复。

## 上线前必须确认(否则服务起不来)

服务器 `.env` 必须有 `SESSION_SECRET` 和 `ID6_SALT`(本地 .env 已有,服务器执行确认):

```bash
# 项目路径从 agent 服务配置读取,无需手写
cd $(systemctl show agent -p WorkingDirectory --value)
grep -E '^(SESSION_SECRET|ID6_SALT)=' .env
```

> 若上面 cd 失败(WorkingDirectory 未配置),用 `sudo find / -maxdepth 4 -name dev_server.py -path '*/scripts/*' 2>/dev/null` 找到项目目录。

缺哪个就补哪个(**ID6_SALT 必须与 Mac 本地 .env 的值一致**,否则 84 名已配身份证的员工全部登不进;SESSION_SECRET 可新生成):

```bash
python3 -c "import secrets;print(secrets.token_hex(32))"
echo 'SESSION_SECRET=上面生成的值' >> .env
```

新守护逻辑:缺这两个变量时 `dev_server.py` 会**拒绝启动**并打印原因(`journalctl -u agent` 可见),不再静默回退到 COS/DeepSeek 密钥。

## 行为变化速查

| 变化 | 影响 |
|---|---|
| 登录限流 | 同 IP 每分钟 8 次 / 每日 150 次;连续失败 5 次/分钟或 60 次/日锁定。真实 IP 取自 X-Forwarded-For(确认 nginx 有 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;`,否则全员共享 127.0.0.1 一个桶) |
| 花名册接口 | GET /api/login 同 IP 每分钟 10 次 / 每日 300 次 |
| 管理员登录 | 必须已配置 id6;普通员工过渡期宽松不变 |
| 停用用户 | 停用立即无法登录(此前留在 users.json 里的可绕过) |
| 遗留哈希 | 若有早期用回退密钥写入的 id6 哈希,登录仍可通过(只读兼容),日志出现 `[IDHASH] 提示` 时在后台重存一次该用户后 6 位即完成迁移 |
| 报错文案 | 登录接口 500 不再返回内部异常细节(细节在 journalctl) |
