## 一、当前形态:四模块 + 管理后台

工作台门户(`/portal.html`)登录后,按组织配置顺序展示四个模块;各页顶栏标签同序切换。

| 顺序 | 模块 | 入口 | 作用 |
|---|---|---|---|
| 1 | **KOS 任务中心** | `/tasks.html` | 领图文素材包、发小红书、回收链接、自发布登记、看本期排行 |
| 2 | **小红书生成助手** | `/index.html` | 一键生成种草/场景/干货/促销文案 + 一键生成封面图 |
| 3 | **销售问答助手** | `/qa.html` | 基于说明书 + 卖点的严格问答 + 销售话术 |
| 4 | **产品资料库** | `/library.html` | 在线下载单页 / 中英文说明书(COS 私有桶) |

管理后台 `/admin.html`(仅管理员):**数据看板 + 用户管理 + 产品管理 + KOS 任务管理 + 操作日志**。

## 二、各模块完整功能

### 1. KOS 任务中心

**素材三分类(管理员上传)**:主图 / 2合1(横版)/ 4合1(竖版),分开上传、分开计数。浏览器预签名直传 COS;进后台自动与桶对账(新文件登记、桶内已删下架),数量与桶实时一致,读不到 COS 时红字提示原因。删库会一并清 COS 目录。

**出图规则(统一 3:4 = 1080×1440,不翻转不变形)**:
- 主图:单张 cover 居中裁成 3:4。
- 2合1:两张横版各裁成 1080×720 后上下拼。
- 4合1:四张竖版各裁成 540×720 后田字 2×2 拼。
- 先把每张裁到精确尺寸再无缝拼接(不二次损失);开图按 EXIF 方向自动转正(修复手机竖拍转向)。

**容量与唯一性**:每份 = 主图×1(随机可复用)+ 一对横版(唯一)+ 一组四张竖版(唯一);单库总量 = min(C(横版数,2), C(竖版数,4)),横版需 ≥2、竖版需 ≥4、主图 ≥1。

**业务闭环(领取页)**:
- 顶部两卡为「本期」口径(周期以最近周四为起点,契合周四发布→下周四回收):卡一「本期任务笔记(已完成/需完成)」、卡二「本期自发布笔记」。
- 领取素材包 → 系统当场挑唯一组合、拉源图、拼图、配文案 → 出 3 图(令牌下载)+ 分栏文案。
- **已领素材包**:每人 1 篇的任务点开直接显示;多篇才显示列表。localStorage 记录可反复查看。
- 文案标题/正文/标签分栏,各自一键复制。
- **回收链接**:只认小红书链接(xhslink.com / xiaohongshu.com),自动从整段口令里抽取纯链接再存(修复跳乱码页问题)。
- **自发布笔记**:业务自发的(非任务)小红书笔记贴链接登记,单独计数并进排行。
- 排行榜按「本期任务完成 + 本期自发布」排名。

**管理端**:建库、发任务(绑库/部门/每人篇数/截止,容量不足预警);数据看板 KOS 标签含笔记排行、每日发布、素材库用量、**任务笔记链接**(可点开看每条已发布笔记)。

### 2. 小红书生成助手

- 选品牌 + 产品后,主卡两个一键按钮:**一键生成文案** / **一键生成封面图**。
- **文案**:四类型(种草/场景/干货/促销);每次随机切换种草角度 + 变体 nonce + 失败重试,标题正文每人不同;正文清洗(去转义/markdown);**标签固定批次**(品牌 + 产品 + 通用标签),永不缺失。
- **封面**(仍需业务上传产品照,主体不变):
  - 「一键导入」智能兜底——有文案按文案出封面文字,无文案按产品资料出。
  - 上传照片 + 三段文字(主标题/副标题/花字正文)→ AI 一次出 3 张候选(3:4、无水印)、失败自动重试一次。
  - 出图规则见第八节。

### 3. 销售问答助手

基于产品说明书 OCR + 卖点资料,三层防幻觉(智能选段 + 强 prompt + 低温度);答主问题后附「销售话术指引」;售后/保修智能兜底(追加 4000480500 热线、按品牌政策回复)。

### 4. 产品资料库

COS 私有桶 + 后端预签名下载(短时效)+ 白名单(「价格与政策」永不外泄);后台「产品管理」可上传/删除/改名资料文件。

## 三、v3.2 本轮更新(2026-07-02 → 07-04)

1. **账号安全升级(P0)**:新增 `lib/session.py`,登录签发 **HMAC-SHA256 签名、24h 有效**的无状态会话 Token;各接口一律从 `Authorization: Bearer` 头取**已认证身份**,不再信任请求体 `_user`,前端无法伪造。密钥读 `SESSION_SECRET`(未配置回退 `DEEPSEEK_API_KEY`)。
2. **KOS 素材三分类 + 出图统一 3:4**(先裁后拼、EXIF 转正、容量重算)。
3. **领取页改「本期」口径**:需完成/已完成卡、自发布笔记(新表 `kos_self_posts`)、已领包每人1篇直接打开、分栏复制、回链只认小红书且只存纯链接;排行本期化。
4. **封面生成升级**:选完产品即可一键出封面;一键导入智能兜底;出图模型升级(中文字更稳);提示词按版式设计思维重做——18 套配色、三级文字一字不差、多色分层 + 关键词高亮、强制简体不出繁体、不加滤镜、照片主体不变。
5. **性能**:服务器改 `ThreadingHTTPServer` 并发 + 接口模块缓存,图片并行加载、领取生成不再卡其他请求;成品图浏览器缓存。
6. **细节**:修复门户刷新闪现历史登录页;前端移除全部大模型技术名词;门户「更新公告」弹窗加入 v3.2。

> v3.0 = SaaS 化地基(统一外壳 + RBAC + 数据收口)+ KOS 任务中心 MVP + 产品资料库。
> v3.1 = 看板重构 + 产品资料管理 + 操作日志 + 每日备份 + KOS 素材直传/自动对账。

## 四、技术栈与架构

| 层 | 选型 |
|---|---|
| 运行 | 多线程 Python `http.server`(`scripts/dev_server.py`,`ThreadingHTTPServer` + 接口模块缓存,自动路由 `/api/<name>`→`api/<name>.py`)|
| 数据库 | 本地 SQLite `data/usage.db`(events / manuals / app_users / app_brands / app_products / library_files / kos_* / admin_audit)|
| 会话 | `lib/session.py`:HMAC-SHA256 无状态签名 Token(24h)|
| 对象存储 | 腾讯云 COS `mkt-1421994952`(ap-shanghai):产品资料 + KOS 素材。下载走后端预签名 GET;上传走浏览器直传预签名 PUT(需子账号写权限 + 桶 CORS)|
| 文案 AI | DeepSeek 系(文案 / 封面字段 / 问答)|
| 视觉 AI | 豆包 Seedream 系(封面出图,模型经 `DOUBAO_MODEL` 环境变量配置)|
| 图像处理 | Pillow(KOS 拼图:三分类、固定 3:4、cover 裁切)|
| 部署 | 腾讯云 Lighthouse + Nginx + systemd `agent.service` + Let's Encrypt |
| 前端 | 原生 HTML + 共享 `app.css`/`app.js`(统一外壳,配置驱动,白标就绪)|

**设计原则**:单进程、单 SQLite、配置驱动、绞杀者式渐进重构(全程不停服)。

> 注:后端与配置里保留真实模型名(便于运维);**前端界面已不出现任何模型名词**。

## 八、出图与文案规则要点

**KOS 拼图**(`lib/image_compose.py`):主图裁一张 3:4;2合1 两横版上下拼;4合1 四竖版田字拼;cover 先裁到单元格精确尺寸再拼;不翻转不变形;EXIF 转正。

**封面出图**(`api/cover-generate.py`):
- 照片主体保持(不加滤镜、不重绘产品/人物)。
- 三级文字:主标题 6-10 字 / 副标题 8-12 字 / 花字正文 8-12 字;严格一字不差、规范简体、禁繁体、画面无其它文字。
- 变化集中在文字的**风格·颜色·字体·排列**:18 套配色方案(白黄经典 / 亮绿 / 白橙 / 撞色卡片 / 黑白+一抹亮色 / 黑金 / 粉白 / 奶茶 / 蓝紫渐变 / 荧光撞色 / 知识卡片 / 蓝白专业 / 绿黑燃脂 / 便签手账 / 红黄撞色 / 深底亮价 / 橙红动感 / 好物+优惠),按文案类型落池,每次随机 3 套并发;多色分层 + 关键词高亮 + 超粗黑体厚描边。
- 封面三段字由 `api/cover-fields.py` + `prompts/封面字段.txt` 提炼。

## 十、待办 / 风险 / 路线图

**待办 / 风险**:
- **COS 密钥轮换**:此前对话中暴露过 SecretKey,建议轮换;轮换后同步更新服务器 `.env` 的 `COS_SECRET_ID/KEY`,否则 KOS 会提示"读不到 COS"。
- 服务器直连 GitHub 常被限速,部署走 ghproxy 镜像(见第七节)。

**增强方向**:
- 封面若要中文 100% 不出错的终极方案:AI 出无字底图 + 服务端 Pillow 叠中文字。
- KOS 真实素材端到端联调;自发布笔记是否设每期上限 / 后台可见列表。

**SaaS 商用化**(有第一个外部客户后再做):多租户隔离(tenant_id 行级)、计费、自助开通、白标域名;后端迁 FastAPI / 数据库迁 Postgres(并发/规模需要时)。
