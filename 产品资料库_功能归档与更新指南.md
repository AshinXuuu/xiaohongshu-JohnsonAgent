> 上线日期:2026-06-17

> 状态:已上线,66 份资料全部可下载 ✅

## 一、功能是怎么跑起来的(一张图看懂)

```
业务点「下载」
     │
     ▼
前端 library.html ──POST──▶ 后端 api/library.py
                              │  1. 查清单白名单(只有清单里的文件能下)
                              │  2. 用 COS 子账号密钥签发一条 5 分钟有效的临时链接
                              │  3. 记一笔 download 事件(用量库)
                              ▼
                       浏览器拿到临时链接 ──▶ 从 COS 私有桶下载
```

- **桶始终私有**,外人直接拿文件 URL 打不开;只有后端签发的临时链接能下,且 5 分钟过期。
- **价格与政策不开放**:清单里根本不包含它,就算有人猜路径也下不到。

## 三、以后更新产品资料怎么做(重点)

更新分三种情况,从简单到复杂:

### 情况 A:给某产品「加一份新资料」或「换一份资料」

最常见。比如 T6 出了新版说明书,或某产品补了张单页。

**第 1 步:传到 COS(位置要对)**
- 进 [COS 控制台](https://console.cloud.tencent.com/cos/bucket) → `mkt-1421994952`
- 进到对应产品目录:`kb` → `原始素材` → `产品库` → 对应品牌 → 对应产品文件夹
- 点「上传文件」把新 PDF 传进去
- 换版的话:把旧文件删掉或覆盖,**文件名尽量和清单保持一致**(见下方命名建议)

**第 2 步:在本地同步一份 + 重建清单**
- 把同一个文件也放进本地 `~/Desktop/红书文案agent/产品库/对应品牌/对应产品/`(保持本地和 COS 一致,方便日后维护)
- 终端跑:
  ```bash
  cd ~/Desktop/红书文案agent
  python3 scripts/gen_library_manifest.py
  ```
  脚本会重新扫描本地产品库,自动更新清单。

**第 3 步:推代码 + 服务器更新**
```bash
# 本地
git add data/library_manifest.json
git commit -m "更新产品资料:XXX"
git push
```
```bash
# 服务器
cd /home/ubuntu/xiaohongshu-JohnsonAgent
git pull
sudo systemctl restart agent
```

**第 4 步:验证**
```bash
set -a && . ./.env && set +a
python3 scripts/verify_cos.py     # 看是否「全部命中」
```

---

### 情况 B:新增一个「整款产品」

- COS:在 `kb/原始素材/产品库/对应品牌/` 下**新建一个产品文件夹**,把单页、中英文说明书传进去
- 本地:在 `~/Desktop/红书文案agent/产品库/对应品牌/` 下同样新建该产品文件夹,放入相同文件
- 然后照「情况 A 的第 2~4 步」走一遍(重建清单 → push → 服务器 pull + 重启 → 验证)

---

### 情况 C:只是「某个文件分类标错了」

比如把中文说明书显示成了英文说明书,不涉及换文件。

- 直接编辑 `data/library_manifest.json`,找到那条,改 `"type"` 字段
- 可选值:`单页` / `中文说明书` / `英文说明书`
- 然后:`git add` → `commit` → `push` → 服务器 `git pull` + `restart`(不用动 COS,也不用验证)

## 四、命名建议(减少对不上的麻烦)

清单是靠**文件名**和 COS 对应的,所以:

- **COS 上的文件名 = 本地文件名**,一字不差(空格、括号、简繁体、日期都算)
- 自动分类靠关键词:文件名带「单页 / 画板」→ 单页;带「ENG / 英文 / Owners Manual」→ 英文说明书;带「说明书 / 簡中 / CHS / OG / Print」→ 中文说明书
- 手动上传时如果图省事改了名,记得本地也改成一样,再重建清单

> 实操更稳的顺序:**先在本地整理好文件 → 重建清单确认分类对 → 再上传 COS(保持同名同结构)**。本地和 COS 始终镜像,就不会对不上。

## 五、出问题怎么排查

| 现象 | 原因 / 解决 |
|---|---|
| verify 报「缺失 N 个」 | COS 上文件名/路径和清单对不上 → 看脚本列出的路径,去 COS 核对文件名 |
| verify 报「AccessDenied」 | 子账号密钥权限不够或失效 → 去 CAM 确认有 COS 只读权限、密钥没被禁用 |
| 页面点下载提示「下载服务未配置」 | 服务器 `.env` 的 `COS_*` 没配全或没生效 → 检查 .env,`restart agent` |
| 某文件下载是旧版 | COS 上还是旧文件 → 重新上传覆盖即可(清单不用动,只要文件名没变) |
